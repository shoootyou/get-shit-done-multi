# Plan 08-03: Platform Documentation

## Objective

Create three platform-focused documentation files that help users understand and choose between Claude Code, GitHub Copilot CLI, and Codex CLI: comparison tables for quick reference, deep dives per platform with real examples, and migration guides for switching platforms.

## Context

**Current state:** Users have no guidance on platform differences, feature compatibility, or how to switch between platforms. The adapter pattern (Phase 3) works but is undocumented.

**Why this matters:** Platform differences affect user experience (tool names, command prefixes, file formats). Clear documentation helps users choose the right platform and migrate when needed.

**From CONTEXT.md:** Three separate platform docs (comparison tables, deep dive, migration FAQ), real frontmatter examples from templates/, cite source file paths, show platform differences side-by-side.

**From RESEARCH.md:** Platform-comparison tables (show-don't-tell pattern), parallel examples for code differences, question-driven migration guide.

**From codebase:** Platform adapters in `bin/lib/platforms/`, templates in `templates/skills/` and `templates/agents/`, adapter transformations in ClaudeAdapter, CopilotAdapter, CodexAdapter.

## Troubleshooting Migration

### Commands not working after switching platforms

**Problem:** Typed `/gsd-help` in Codex (should be `$gsd-help`)

**Solution:** Remember command prefix:
- Claude & Copilot: `/gsd-*`
- Codex: `$gsd-*`

### Skills not loading on new platform

**Problem:** Installed GSD but commands not recognized.

**Solution:**
1. Verify installation:
   \`\`\`bash
   npx get-shit-done-multi --version
   \`\`\`
2. Check files exist:
   \`\`\`bash
   ls ~/.claude/skills/gsd-*  # Claude
   ls ~/.copilot/skills/gsd-*  # Copilot
   ls ~/.codex/skills/gsd-*  # Codex
   \`\`\`
3. Restart AI assistant:
   - Claude: Restart editor
   - Copilot: Close terminal, open new one
   - Codex: Run `codex reload`

### .planning/ directory not working after migration

**Problem:** State files show wrong platform or commands fail.

**Diagnosis:**
\`\`\`bash
# Check .planning/ directory
cat .planning/STATE.md
# Should NOT mention specific platform (platform-agnostic)
\`\`\`

**Solution:**
- `.planning/` is platform-agnostic by design
- If references to platform exist, they're informational only
- Commands work the same (just use correct prefix)

**Solution:**
\`\`\`bash
# Check versions
npx get-shit-done-multi --version

# Upgrade old platform to match
