## What Happens During Installation

1. **Pre-flight validation:**
   - Check Node.js version (‚â•20)
   - Verify available disk space
   - Test write permissions
   - Validate paths for security

2. **Platform detection:**
   - Scan for existing GSD installations
   - Detect available CLI binaries
   - Check installation paths

3. **Template processing:**
   - Load templates from package
   - Replace variables ({{PLATFORM_ROOT}}, {{COMMAND_PREFIX}})
   - Transform tool names per platform
   - Adjust frontmatter for platform specs

4. **File operations:**
   - Create directories (skills/, agents/, get-shit-done/)

   - Copy shared resources (references, workflows)

5. **Post-installation:**
   - Generate manifest (.gsd-install-manifest.json)
   - Record installation metadata (version, platform, timestamp)
   - Display success message with next steps

## Troubleshooting

### Error: "Node.js 20 or higher required"

**Cause:** Your Node.js version is below 20 (Node 16 is EOL).

**Solution:**
\`\`\`bash
# Check current version
node --version

# Upgrade with nvm (recommended)
nvm install 20
nvm use 20

# Or with Homebrew (macOS)
brew install node@20
\`\`\`

### Error: "Permission denied"

**Cause:** No write permission to target directory.

**Solutions:**

1. **Use local installation** (recommended):
   \`\`\`bash
   npx get-shit-done-multi --local
   \`\`\`

2. **Fix permissions** (global installation):
   \`\`\`bash
   # macOS/Linux
   sudo npx get-shit-done-multi --global
   \`\`\`

3. **Change ownership** (if directory exists):
   \`\`\`bash
   # macOS/Linux
   sudo chown -R $USER ~/.claude ~/.copilot ~/.codex
   \`\`\`

### Error: "Platform not detected"

**Cause:** No supported AI platform installed, or platform not in PATH.

**Solutions:**

1. **Install a supported platform:**
   - Claude Code: https://code.claude.com/
   - GitHub Copilot CLI: `gh extension install github/gh-copilot`
   - Codex CLI: [Install link]

2. **Verify platform in PATH:**
   \`\`\`bash
   # Check which platforms are available
   which claude
   which copilot
   which codex
   \`\`\`

3. **Use explicit flag:**
   \`\`\`bash
   # Install anyway (platform detection can be skipped)
   npx get-shit-done-multi --claude --yes
   \`\`\`

### Error: "Installation already exists"

**Cause:** GSD is already installed at target location.

**Solutions:**

1. **Upgrade instead:**
   \`\`\`bash
   # Upgrade prompts automatically in interactive mode
   npx get-shit-done-multi
   \`\`\`
   See [how-to-upgrade.md](how-to-upgrade.md) for details.

2. **Uninstall first:**
   \`\`\`bash
   # Remove existing installation
   rm -rf ~/.claude/skills/gsd-*
   rm -rf ~/.claude/agents/gsd-*
   rm -rf ~/.claude/get-shit-done

   # Then reinstall
   npx get-shit-done-multi
   \`\`\`
   See [how-to-uninstall.md](how-to-uninstall.md) for safe removal.

# How to Upgrade GSD

Guide to upgrading Get Shit Done to newer versions.

## Quick Start

Run the installer again:

\`\`\`bash
npx get-shit-done-multi
\`\`\`

If a newer version exists, you'll be prompted to upgrade.

---

## How Upgrade Detection Works

GSD uses installation manifests to track versions:

1. **Manifest location:**
   - Claude: `~/.claude/get-shit-done/.gsd-install-manifest.json`
   - Copilot: `~/.copilot/get-shit-done/.gsd-install-manifest.json` (global) or `.github/get-shit-done/.gsd-install-manifest.json` (local)
   - Codex: `~/.codex/get-shit-done/.gsd-install-manifest.json`

2. **Version comparison:**
   - Installer reads manifest, compares with npm registry version
   - Uses semver for version comparison (2.1.0 > 2.0.0)
   - Prompts user if newer version available

3. **Multiple installations:**
   - Each platform tracked separately
   - Global and local tracked separately
   - Can have different versions on different platforms

---

# Troubleshooting

Common issues and solutions for GSD installation and usage.

## Installation Issues

### Error: "Node.js 20 or higher required"

**Symptoms:**
- Installation fails immediately
- Error message: "Node.js 20 or higher required"
- Shows current Node version

**Cause:**
Your Node.js version is below 20. Node 16 reached EOL (end-of-life) and has security vulnerabilities.

**Solutions:**

1. **Check current Node version:**
   \`\`\`bash
   node --version
   # If shows v16.x.x or v18.x.x, upgrade needed
   \`\`\`

2. **Upgrade with nvm (recommended):**
   \`\`\`bash
   # Install nvm if not installed
   curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

   # Install Node 20
   nvm install 20
   nvm use 20
   nvm alias default 20

   # Verify
   node --version  # Should show v20.x.x
   \`\`\`

3. **Upgrade with Homebrew (macOS):**
   \`\`\`bash
   brew install node@20
   brew link node@20
   \`\`\`

4. **Download from nodejs.org:**
   - Visit https://nodejs.org/
   - Download Node 20 LTS installer
   - Run installer
   - Verify with `node --version`

---

### Error: "Permission denied"

**Symptoms:**
- Installation fails during file operations
- Error: "EACCES: permission denied, mkdir '/Users/.../.claude/skills'"
- Occurs on macOS or Linux

**Cause:**
No write permission to target directory (usually global installation path).

**Solutions:**

1. **Use local installation (recommended):**
   \`\`\`bash
   npx get-shit-done-multi --local
   \`\`\`
   Local installation writes to current project, no special permissions needed.

2. **Run with sudo (global installation):**
   \`\`\`bash
   sudo npx get-shit-done-multi --global
   \`\`\`

3. **Fix directory permissions:**
   \`\`\`bash
   # Change ownership to your user
   sudo chown -R $USER ~/.claude ~/.copilot ~/.codex

   # Then install normally
   npx get-shit-done-multi
   \`\`\`

4. **Create directories first:**
   \`\`\`bash
   mkdir -p ~/.claude/skills ~/.claude/agents ~/.claude/get-shit-done
   chmod 755 ~/.claude ~/.claude/skills ~/.claude/agents
   npx get-shit-done-multi --claude
   \`\`\`

   which copilot  # Should show path
   \`\`\`

   **Codex CLI:**
   - Visit [Codex CLI website]
   - Follow installation instructions
   - Verify: `which codex`

2. **Add platform to PATH:**
   \`\`\`bash
   # Find where platform is installed
   find ~ -name "claude" -o -name "copilot" -o -name "codex"

   # Add to PATH in ~/.zshrc or ~/.bashrc
   export PATH="/path/to/platform/bin:$PATH"

   # Reload shell
   source ~/.zshrc
